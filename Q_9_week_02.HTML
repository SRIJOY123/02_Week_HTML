1. <h1> to <h6>
    • Used to define headings of different levels, with <h1> being the highest and <h6> the lowest.
    • Best practice is to use <h1> for the main heading of the page and use the subsequent levels for subheadings.
    • Headings should be used hierarchically and reflect the structure of the content.
  2. <p>
    • Used to define a paragraph of text.
    • Best practice is to separate paragraphs with empty lines to enhance readability.
    • Avoid using multiple <p> tags for styling purposes; use CSS for formatting instead.
  3. <em>
    • Used to emphasize or highlight text.
    • Best practice is to use <em> for semantic emphasis, not just for visual styling.
    • Screen readers and search engines may interpret the emphasized text differently, so use it appropriately.
  4. <strong>
    • Used to indicate strong importance or significance.
    • Best practice is to use <strong> for semantic emphasis, not just for visual styling.
    • Screen readers and search engines may interpret the strong text differently, so use it appropriately.
  5. <ul> and <li>
    • <ul> is used to create an unordered (bullet) list.
    • <li> is used to define each item in the list.
    • Best practice is to use unordered lists when the order of items doesn't matter.
  6. <ol> and <li>
    • <ol> is used to create an ordered (numbered) list.
    • <li> is used to define each item in the list.
    • Best practice is to use ordered lists when the order of items is important.
  7. <a>
    • Used to create a hyperlink or anchor link.
    • Best practice is to provide a meaningful and descriptive text for the link.
    • Use the href attribute to specify the target URL or location.
  8. <img>
    • Used to embed an image into the web page.
    • Best practice is to provide alternative text using the alt attribute for accessibility.
    • Specify the image source using the src attribute.
